# AI 辅助研究指南

> 来源：肖璟《如何快速了解一个行业》第13章

## AI 在研究中的角色定位

**书中核心观点**：把 AI 当作"刚毕业的优秀大学生"——需要清晰明确的指令，需要验证输出结果，能高效完成执行层面的工作但最终判断仍需你来把关。

## AI 在当前阶段（2026年中）的能力图谱

### 主流大语言模型

| 模型 | 类型 | 特点 |
|------|------|------|
| **Claude** (Anthropic) | 闭源 | 长文本分析、结构化写作、代码生成、安全对齐 |
| **GPT-5.2** (OpenAI) | 闭源 | 通用能力强、多模态、深度研究 |
| **Gemini** (Google) | 闭源 | 多模态、与Google生态整合 |
| **DeepSeek** | 开源 | 性价比高、中文能力强、推理能力突出 |
| **通义千问** (阿里) | 开源 | 中文能力强、阿里云生态 |
| **文心一言** (百度) | 闭源 | 中文、百度生态 |
| **Kimi** (月之暗面) | 闭源 | 超长上下文、文档分析 |
| **GLM** (智谱AI) | 开源 | 国产自主、多模态 |

### AI 的三大局限（书中原文）

1. **回答不稳定**：不总是生成相同结果 → 可通过降低温度值(0-0.3)提高稳定性
2. **AI幻觉**：会生成与事实不符的内容 → 必须交叉验证，不要盲信AI的"事实陈述"
3. **知识滞后**：训练数据有截止日期 → 需要联网搜索或RAG补充最新信息

## 研究各阶段的 AI 应用

### 第一阶段：备菜（资讯输入）🔍

| 应用场景 | 怎么做 | 推荐工具 |
|---------|--------|---------|
| **假设拆解** | 将研究问题拆为MECE议题树 | Claude、ChatGPT |
| **搜索验证** | 搜索相关数据和指标，标注来源 | 带联网搜索的AI（Claude WebSearch、Perplexity） |
| **拓宽观点** | 让AI扮演不同立场，完整阐述双方观点 | Claude（提示"请从正方和反方两个角度分析"） |
| **文档总结** | 快速提炼长文档要点 | Kimi、Claude（上传PDF/网页链接） |
| **数据录入** | OCR识别图片/PDF中的表格，转结构化数据 | AI+OCR工具 |
| **音视频转文字** | 会议录音、访谈转文字后分析 | 飞书妙记（区分发言人）、剪映（智能字幕） |

### 第二阶段：烹饪（资讯处理）🍳

| 应用场景 | 怎么做 | 注意事项 |
|---------|--------|---------|
| **罗列事实** | 让AI帮你按议题整理所有收集到的数据 | 原始数据需要你先提供给AI |
| **寻找规律** | 让AI分析数据中的趋势和关联 | AI能找到统计意义上的关联，但不一定理解因果关系 |
| **提炼要点** | 用金字塔原理提示AI：结论→论据→数据 | 明确要求"结论先行" |
| **高级数据分析** | 上传数据集，让AI编写代码分析 | ChatGPT Advanced Data Analysis、Claude Analysis |
| **模型构建** | 让AI构建金融预测模型 | 书中案例：随机森林预测贷款审核，准确率78% |

### 第三阶段：摆盘（资讯输出）📊

| 应用场景 | 怎么做 | 推荐工具 |
|---------|--------|---------|
| **辅助写作** | 按SCQR结构组织内容，AI辅助润色 | Claude、ChatGPT |
| **PPT生成** | 从文本大纲生成幻灯片 | Gamma.ai、WPS AI、AiPPT.cn |
| **图表生成** | 描述数据让AI生成可视化 | Claude Artifacts、ChatGPT Code Interpreter |
| **邮件/简报** | 将长报告压缩为执行摘要 | Claude |

## AI 使用最佳实践（来自书中+当前最佳实践）

### 提示词设计原则

1. **明确角色**："你是一位资深的行业分析师……"
2. **明确任务**："请分析预制菜行业的商业模式……"
3. **明确格式**："请用表格形式输出，包含以下列……"
4. **明确标准**："请标注每个数据来源，优先使用2025-2026年的数据"
5. **分步引导**：复杂任务拆成多轮对话，不要一次问太多

### 验证原则

1. **任何AI给出的"事实"都要交叉验证**——特别是数字、日期、公司名
2. **让AI标注来源**——如果AI说"据XX机构数据"，去原始来源核实
3. **多AI交叉验证**——同一个问题问2个不同AI，看答案是否一致
4. **逻辑审查**——AI的推理链条有没有跳跃？结论是否真的被论据支撑？

### 迭代优化

- 第一轮给大致方向 → 看结果 → 发现问题 → 第二轮细化提示词 → 再看结果
- 通常2-3轮可以得到满意结果
- 如果3轮后仍不满意，可能是任务本身不适合AI（换工具或换方法）

## 2025-2026 年 AI 研究新趋势

书中写于2024年末，以下补充当前（2026年中）的新发展：

1. **AI Agent 自主研究**：从被动问答到主动研究，Agent可以自主规划搜索步骤
2. **深度研究（Deep Research）**：多轮搜索→交叉验证→综合报告，接近人类研究员的完整流程
3. **实时数据获取**：AI联网能力普及，时效性问题大幅改善
4. **多模态分析**：可以直接读取图表、分析视频内容
5. **本地知识库（RAG）**：企业可以将内部数据与AI结合
