## 商业模式画布（9要素）

来源：《商业模式新生代》(Business Model Generation)

| 模块 | 说明 | 关键问题 |
|------|------|---------|
| **客户细分** | 核心客户群是谁？ | 谁是你的付费用户？他们有什么特征？ |
| **价值主张** | 解决什么痛点？ | 用户为什么选择你而非竞品？ |
| **渠道通路** | 如何触达用户？ | 线上/线下？直销/经销？ |
| **客户关系** | 如何维护关系？ | 自助服务？专属客服？社群运营？ |
| **收入来源** | 靠什么赚钱？ | 卖产品？订阅费？广告？抽佣？ |
| **核心资源** | 最重要的资产？ | 技术？品牌？数据？人才？牌照？ |
| **关键业务** | 日常在做什么？ | 研发？生产？营销？运营？ |
| **重要伙伴** | 关键合作伙伴？ | 供应商？渠道商？技术伙伴？ |
| **成本结构** | 钱花在哪里？ | 固定成本 vs 变动成本？最大的成本项？ |

## 商业模式验证：对标法

### 时间对标法

**逻辑**：人类需求是永恒的，变的只是满足需求的方式。

操作方法：往前回溯，这个需求在历史上是如何被满足的？

| 需求 | 历史满足方式 | 现代满足方式 |
|------|-------------|-------------|
| 在家快速搞定一顿饭 | 干粮→方便面→电话外卖 | 外卖平台、预制菜 |
| 出行 | 走路→轿子/马车 | 汽车、网约车、共享单车 |
| 获取信息 | 口口相传→报纸→广播→电视 | 互联网、社交媒体、AI问答 |
| 社交 | 信件→电报→电话→短信 | 微信、视频通话、社交App |

**验证逻辑**：如果历史上这个需求一直存在且在增长，那么新的满足方式大概率有市场。

**反例**：共享篮球——篮球本身不贵不占地，打篮球的人不会缺篮球，历史上也几乎没有"篮球租赁"这种需求，所以是伪需求。

### 空间对标法（时光机理论）

**逻辑**：人类的底层需求是相通的，不同市场的消费行为会"重演"。

操作方法：对标成熟市场，看是否有类似模式？

经典路径：
- **C2C (Copy to China)**：ICQ→QQ、雅虎→搜狐、谷歌→百度、 Amazon→淘宝、Uber→滴滴
- **现在正在复制到东南亚**：Shopee（复制淘宝）、Grab（复制滴滴+美团）
- **CFC (Copy from China)**：TikTok 出海、SHEIN 快时尚出海、Temu 拼多多出海

**验证逻辑**：如果在成熟市场已被验证，那么在新兴市场大概率也能成立（但需考虑本地化差异）。

## 需求排序矩阵

对需求进行"频次 × 弹性"交叉评估：

| | 弹性小（刚需） | 弹性大（非刚需） |
|--|--------------|----------------|
| **频次高** | ⭐⭐⭐⭐⭐ 吃饭（最好） | ⭐⭐⭐ 吃猪肉（可替代） |
| **频次低** | ⭐⭐⭐ 婚庆（获客成本高） | ⭐ 景点骑马（最差） |

**核心结论**：频次高、弹性小，二者至少取其一才可能有不错的收入。

## 利润可行性评估

### 定性分析：收入层面

- **频次**：被使用的频次越高越好（高频打低频）
- **弹性**：需求量对价格变动越不敏感越好（刚需）

### 定性分析：成本层面

- **标准化程度高** → 综合成本低，可规模化（海底捞选火锅而非炒菜）
- **标准化程度低** → 可以卖高价但难规模化（麦肯锡咨询 vs SaaS软件）

### 定量分析：UE 单位经济模型

**"最小运作单位"** 的概念：

| 行业 | 最小运作单位 |
|------|-------------|
| 餐饮 | 单店（一家门店） |
| 网约车 | 一辆车 |
| 美妆 | 一种单品（SKU） |
| 外卖 | 一个订单 |
| SaaS | 一个客户 |
| 电商 | 一个订单 |
| 连锁零售 | 单店 |

**核心逻辑**：如果最小运作单位亏钱，规模化复制也很难赚钱；反之则可以扩张。

**UE 模型模板**：
```
收入端：
  客单价/单均收入 × 日均/月均单量 = 日均/月均收入

成本端：
  变动成本（原材料、物流等）
  + 固定成本（租金、人工、折旧等）
  = 总成本

利润端：
  收入 - 成本 = 单位利润
  毛利率 = (收入 - 变动成本) / 收入
  净利率 = 单位利润 / 收入
  回本周期 = 初始投入 / 月均利润
```

**海底捞案例**（2019年）：
- 75张桌子，每桌3.5人，翻台率4.8次/天 → 日均1260人次
- 客单价105.2元 → 日营业额约13.26万
- 年营业360天 → 营收约4772万
- 单店年经营利润约888万

**核心指标**：翻台率（4次/天为管理层接受底线）

### 终局视角

即使当前UE模型亏损，重点在于未来有没有改善的可能。通常建立3个UE模型：
1. **当前模型**：反映实际情况
2. **盈亏平衡模型**：什么条件下可以打平？
3. **终局模型**：稳态下的理想情况

## 无法收费的商业模式

以微信为代表：
- 免费通信服务，收入来源为广告主和商家（ARPU模式）
- 导入期甚至成长期难以准确量化
- 容易导致估值泡沫（如2015年互联网泡沫）
