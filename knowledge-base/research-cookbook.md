# 资讯料理法与研究输出框架

> 来源：肖璟《如何快速了解一个行业》第10-11章

## 研究的本质（第10章）

### 研究为什么能创造财富？

研究 = 降低信息不对称 → 做出更好的决策 → 创造价值（投资赚钱、企业决策、职业选择）

### DIKW 模型

资讯处理的四个层次：

```
数据 (Data)
  ↓ 结构化、清洗
信息 (Information)
  ↓ 分析、建立联系
知识 (Knowledge)
  ↓ 内化、形成判断力
智慧 (Wisdom)
```

- **数据**：原始事实和数字（"今年GDP增长5.0%"）
- **信息**：经过整理、可理解的数据（"GDP增速比去年下降0.2个百分点"）
- **知识**：信息之间的联系和规律（"增速下降主要是房地产拖累"）
- **智慧**：能做出正确判断和决策（"当前环境下应该减配地产股，增配消费股"）

### 事实判断 vs 价值判断

| | 事实判断 | 价值判断 |
|--|---------|---------|
| 问题 | "是什么" | "好不好" |
| 性质 | 客观，可验证 | 主观，因人而异 |
| 示例 | "茅台PE是35倍" | "35倍PE贵不贵" |
| 陷阱 | 数据可能不准确 | 容易被自身立场左右 |

研究的核心：**先做好事实判断，再基于事实做价值判断**。不要在事实不清楚的时候急于下结论。

### 假设驱动

**不要盲目收集信息，先提出假设，再找证据验证。**

```
传统方式：广泛收集信息 → 在海量信息中迷失 → 不知道该关注什么
假设驱动：提出假设 → 寻找证据验证/证伪 → 聚焦、高效
```

## 资讯料理法（第11章）

全书的研究方法论核心，分三个阶段：

### 第一阶段：备菜（资讯输入）

**目标**：高效收集相关资讯，不遗漏关键信息

核心方法：
1. **假设驱动**（见上文）——先用MECE拆解问题，再按议题收集
2. **信息分层**：
   - 第一层：宏观数据和行业概况（统计机构、行业报告）
   - 第二层：竞争格局和商业模式（券商研报、公司公告）
   - 第三层：一手验证和深度洞察（专家访谈、草根调研）
3. **警惕信息茧房**：主动寻找反面观点，刻意接触不同立场的信息源

### 第二阶段：烹饪（资讯处理）

**目标**：从信息中提炼知识和洞见

核心方法：
1. **罗列事实** → 把数据和信息按议题整理成清单
2. **寻找规律** → 事实之间有什么关联？有什么趋势？
3. **提炼要点** → 用金字塔原理组织：核心结论是什么？支撑论据是什么？
4. **交叉验证** → 不同来源的数据一致吗？不一致的原因是什么？

### 第三阶段：摆盘（资讯输出）

**目标**：清晰、有力地将研究成果传达给受众

核心方法：
1. **SCQR框架**（见下文）
2. **金字塔原理**（见下文）
3. **图表辅助**：书中含200+张图表，好的图表胜过千言
4. **针对受众调整**：给CEO看结论，给分析师看数据，给大众看故事

## SCQR 框架（"空·雨·湿·伞"）

麦肯锡经典的故事叙述框架：

| 英文 | 中文 | 含义 | 示例 |
|------|------|------|------|
| **S**ituation | 空（情境） | 天空是什么样的？交代背景 | "中国咖啡市场过去5年增长了5倍" |
| **C**omplication | 雨（冲突） | 发生了什么变化？制造张力 | "但人均咖啡消费量仍只有日本的1/10" |
| **Q**uestion | 湿（问题） | 面临什么决策？提出待解决的问题 | "中国咖啡市场还有多大增长空间？" |
| **R**esolution | 伞（方案） | 应该怎么做？给出答案和建议 | "预计还有5-8倍增长空间，建议关注下沉市场机会" |

**SCQR的使用原则**：
- 先说结论（R），再展开S→C→Q→R的完整叙述
- S和C要具体、有数据支撑
- Q要把问题聚焦到可回答的范围
- R要明确、可执行

## 金字塔原理

巴巴拉·明托提出的结构化表达方法：

```
           ┌──────────────┐
           │   核心结论    │  ← 先说结论
           └──────┬───────┘
      ┌───────────┼───────────┐
  ┌───┴───┐   ┌───┴───┐   ┌───┴───┐
  │ 论据1  │   │ 论据2  │   │ 论据3  │  ← 支撑论据
  └───┬───┘   └───┬───┘   └───┬───┘
  ┌───┴───┐   ┌───┴───┐   ┌───┴───┐
  │数据/  │   │数据/  │   │数据/  │   ← 具体数据
  │事实   │   │事实   │   │事实   │
  └───────┘   └───────┘   └───────┘
```

金字塔原理四原则：
1. **结论先行**：先说最重要的结论
2. **以上统下**：上层概括下层
3. **归类分组**：相似的内容放在一起
4. **逻辑递进**：按时间/结构/程度排序

## MECE 原则

**Mutually Exclusive（相互独立）+ Collectively Exhaustive（完全穷尽）**

拆解问题时不重不漏：
- 相互独立：每个子问题之间不要有重叠
- 完全穷尽：所有子问题加起来覆盖了整个问题

**常见的MECE拆解维度**：
- 按时间：过去→现在→未来
- 按结构：上游→中游→下游
- 按逻辑：内因→外因
- 按要素：人→货→场
- 按公式：利润 = 收入 - 成本 = 量 × 价 - 固定成本 - 变动成本
