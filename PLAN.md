# 行业/企业分析工具——工作计划

## Context

**用户**：行业协会工作人员，有 ECS 服务器（已有 2 个 Streamlit 应用），vibe-coder 风格，中文交互偏好。

**目标**：基于肖璟《如何快速了解一个行业》(2025, 人邮社, 332页) 的完整方法论，构建可用的行业/企业分析工具。形式为 **先 Claude Code Skills（快速可用）→ 后 Streamlit 平台（完整展示）** 的混合方案。

**用户五点要求**：
1. 明确 7 大维度具体内容
2. 输出成果中补充应用的模型和参考资源
3. 所有数据必须是最新的，不得使用过时数据
4. 新增企业对比功能
5. **所有数据来源必须标注具体网址（URL）**，不得只写机构名称；优先使用书中推荐的资源（统计机构/研报/数据库/另类数据/专家网络/公司公开信息）

## 七大分析维度详解 + 模型 + 资源（第2-8章）

### 维度一：可行性 —— 商业模式跑得通吗？
| 模型 | 说明 |
|------|------|
| 商业模式画布 | 9 要素拆解：客户细分、价值主张、渠道、客户关系、收入来源、核心资源、关键业务、重要伙伴、成本结构 |
| 时间对标法 | 往前回溯历史上是否有类似需求（需求永恒，服务形式可变） |
| 空间对标法 | 对标成熟市场是否有类似案例（时光机理论、C2C 复制模式） |
| UE 模型 | 最小运作单位（单店/单客/单车/单品）的收入-成本-利润结构 |
| 需求排序矩阵 | 频次 × 弹性：频次高+弹性小=最优，频次低+弹性大=最差 |
| **参考资源** | 《商业模式新生代》、各公司招股书 |

### 维度二：规模性 —— 市场天花板多高？
| 模型 | 说明 |
|------|------|
| TAM/SAM/SOM | 总可寻址市场→可服务市场→可获取市场，逐层收窄 |
| 自上而下法 | 宏观数据拆解：总人口 × 渗透率 × 消费频次 × 客单价 |
| 自下而上法 | 微观加总：门店数 × 单店营收 |
| 渗透率判断法 | <15%导入期 / 15-35%成长期 / 35%+成熟期 |
| **参考资源** | 国家统计局、行业协会报告、券商研报、招股书行业章节 |

### 维度三：防守性 —— 护城河够深吗？
| 大类 | 子项 |
|------|------|
| 资源垄断（生产要素） | 资金/资本、人才/技术/专利、牌照/准入、数据、自然资源 |
| 关系垄断（生产关系） | 网络效应、规模效应、转换成本、品牌/渠道 |
| **参考资源** | Pat Dorsey《巴菲特的护城河》、波特五力模型、行业 CRn 数据 |

### 维度四：盈利性 —— 钱被谁赚走了？
| 模型 | 说明 |
|------|------|
| 横向格局 | CR4/CR8 行业集中度，判断分散还是集中 |
| 纵向格局 | 产业链各环节毛利率/净利率对比，找出利润最丰厚环节 |
| 五力模型 | 供应商议价力、买方议价力、新进入者威胁、替代品威胁、现有竞争者 |
| 赔率 vs 概率 | 龙头提升市占率=高概率，分散行业中的黑马=高赔率 |
| **参考资源** | 上市公司年报分业务毛利率、券商产业链深度报告、Wind/东方财富 CRn |

### 维度五：估值 —— 当前阶段值多少钱？
| 生命周期 | 适用估值方法 |
|---------|-------------|
| 导入期 | VC 估值法（市场规模反推、可比公司对标） |
| 成长期 | PEG、PS（市销率） |
| 成熟期 | PE（市盈率）、PB（市净率）、EV/EBITDA |
| 衰退期 | 资产重估、清算价值 |
| 通用公式 | 公司市值 = 净利润 × 市盈率（PE） |
| **参考资源** | Damodaran 各行业估值倍数、券商估值报告可比公司对比表 |

### 维度六：外部因素 —— PEST 分析
| 维度 | 关键检查项 |
|------|-----------|
| 政治 (P) | 产业政策、监管法规、税收优惠/限制、贸易政策、反垄断 |
| 经济 (E) | GDP 增速、居民收入、利率、通胀、汇率、消费信心 |
| 社会文化 (S) | 人口结构、消费观念变迁、生活方式变化、教育水平 |
| 技术 (T) | 关键技术突破、技术成熟度（Gartner 曲线）、数字化、AI 影响 |
| 技术创新金字塔 | 基础设施→硬件→操作系统→应用程序→生态参与者 |
| **参考资源** | 国务院/部委政策、Gartner Hype Cycle、国家统计局/央行经济数据 |

### 维度七：景气度 —— 当前行业冷热如何？
| 行业类型 | 核心景气度指标 |
|---------|---------------|
| 上游原材料 | 价格（大宗商品）、库存 |
| 中游制造业 | 产销量、产能利用率、PMI |
| 必选消费 | CPI、社零数据 |
| 可选消费 | 房地产（后周期）、居民收入预期 |
| 服务业 | 宏观经济、PMI 服务业指数 |
| TMT/新兴行业 | 自身产业周期（出货量、渗透率） |
| **两种方法** | ①产业链上下游推导（找核心驱动环节）②关键指标高频跟踪 |
| **参考资源** | 国家统计局高频数据、行业协会月度/季度数据、另类数据（卫星/招聘/交通/电商评论）、Wind 终端 |

## 数据获取策略（API + 搜索混合）

**原则：所有数据必须最新。** 每次分析时通过以下渠道实时获取：

| 渠道 | 数据类型 |
|------|---------|
| 国家统计局 API | GDP、CPI、PMI、人口、社零等 |
| 东方财富/Choice | 上市公司财务、行情、行业分类 |
| 行业协会官网 | 细分行业统计数据 |
| WebSearch 兜底 | 最新券商研报、政策动态、公司财报、景气度指标 |

**时效性要求**：市场规模≤1年 / 财务数据→最新财报期 / 政策→最新发布 / 景气度指标→按月/周更新

## Phase 1: Claude Code Skills（6 个，1-2 周）

### Skill 1: `/行业分析 <行业名>`
**对应章节**：第1-8章

**工作流程**：
1. 实时搜索最新数据（市场规模、渗透率、政策、头部公司）
2. 渗透率法判断生命周期阶段
3. 七维度逐一分析（按生命周期调整权重）
4. 每维度标注数据来源 URL 和时间
5. SCQR 格式输出报告

**应用的模型**：产业生命周期 + 商业模式画布 + 对标法 + TAM/SAM/SOM + UE 模型 + 护城河分类 + CRn + 五力模型 + PEST + 景气度指标体系 + SCQR

**输出**：3000-5000 字报告，含生命周期判断 + 七维度分析 + 关键指标当前值 + 风险评估 + 结论建议

### Skill 2: `/企业分析 <企业名>`
**对应章节**：第2-6章

**工作流程**：
1. 实时搜索企业最新财报、业务数据
2. 行业定位 + 生命周期判断
3. 商业模式画布绘制（9 要素）
4. UE 模型构建（填入具体数字）
5. 护城河评分（各项 1-5 分 + 总分）
6. 竞争地位 + 估值框架

**应用的模型**：商业模式画布 + 对标法 + UE 模型 + 护城河评分卡 + 五力模型 + 估值阶段对照表

**输出**：画布 9 要素表 + UE 模型数值 + 护城河评分卡 + 竞争定位 + 估值框架

### Skill 3: `/企业对比 <企业A> <企业B>` **[新增]**
**对应章节**：第2-6章（企业横向对比）

**工作流程**：
1. 同时搜索两企业最新数据
2. 商业模式画布并排对比
3. UE 模型关键指标对比（客单价、毛利率、净利率、回本周期）
4. 护城河 PK（各项评分对比）
5. 竞争格局定位 + 估值对比
6. 综合判效

**应用的模型**：商业模式画布 + UE 模型 + 护城河评分 + 估值对照 + 对标法 + 五力模型

**输出**：并排画布对比 + UE 指标 PK 表 + 护城河评分 PK + 估值对比 + 综合结论及选择建议

### Skill 4: `/行业对比 <行业A> <行业B>`
**对应章节**：第1-8章（跨行业）

**工作流程**：
1. 同时搜索两行业最新数据
2. 生命周期并排定位
3. 七维度并行对比（每维度并排呈现两行业最新数据）
4. 差异高亮 + 选择建议

**应用的模型**：产业生命周期 + 七大维度全部 + 对标法 + 护城河分类 + PEST

**输出**：并排对比表 + 七维度差异分析 + 选择建议（标注所有数据来源和时间）

### Skill 5: `/研究方法 <问题>`
**对应章节**：第10-13章

**工作流程**：
1. 假设驱动：MECE 议题树拆解
2. 资讯来源推荐（匹配书中数据源分类，给出具体来源名称和链接）
3. AI 工具建议（根据第13章适配）
4. 资讯料理法步骤（备菜→烹饪→摆盘）

**应用的模型**：假设驱动 + MECE + 资讯料理法 + DIKW 模型 + SCQR + 金字塔原理

**输出**：议题树 + 数据源清单 + AI 工具建议 + 研究步骤

### Skill 6: `/术语解释 <术语>`
**对应章节**：附录（100+ 名词）

**工作流程**：查附录定义 → 书中上下文详解 → 关联概念 → 分析中使用示例

**输出**：术语定义 + 书中出处 + 关联概念 + 使用示例

## Phase 3: Streamlit 平台（2-3 周）

**定位**：ECS 上的行业研究工作站，端口 8513

**技术栈**：Streamlit + Anthropic API + Plotly + python-docx + SQLite + Docker

### 6 个模块：

| 模块 | 功能 |
|------|------|
| **行业分析工作台** | 输入行业→自动分析→可视化（生命周期图/五力图/产业链利润图/PEST 雷达图）→报告导出 |
| **企业分析工作台** | 商业模式画布交互填写 + UE 计算器 + 护城河评分雷达图 + 报告导出 |
| **企业对比工作台** | 双企业并排输入→画布/UE/护城河/估值逐项 PK 可视化 |
| **行业对比工作台** | 双行业并排输入→七维度并排+差异高亮 |
| **研究工具箱** | 议题树生成 + 数据源推荐 + AI 研究向导 |
| **报告中心** | 历史记录 + 模板管理 + PDF/Word/Markdown 导出 |
